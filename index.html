Data Health

데이터 신선도와 실패 상태

정적 페이지는 생성된 JSON만 읽습니다. 최신 데이터는 refresh 스크립트 또는 GitHub Actions에서 갱신합니다.

Refresh

새 티커 종가 추가·업데이트

브라우저는 외부 금융 API를 직접 호출하지 않습니다. 한국 ETF 6자리 코드(예: 069500, 0167A0)는 .KS로 정규화하고, 새 ETF는 종가와 holdings 파싱 대상에 함께 넣습니다. 로컬 npm run dev에서는 티커 입력만으로 refresh/test가 자동 실행됩니다.

Actions에서 실행
npm run refresh:data && npm test

로컬 자동 갱신은 npm run dev로 연 페이지에서만 동작합니다. 공개 Pages는 보안상 브라우저에서 GitHub Actions write token을 받지 않으므로, 새 티커가 감지되면 입력값과 명령을 자동으로 준비하고 Actions 링크/명령 복사로 갱신합니다. 갱신 후 생성 JSON을 다시 읽으면 보유 주수 계산에 실제 종가가 사용됩니다.

TSLL/SNXX/0167A0/RAM ETF 갱신 입력과 자동 refresh 감지를 준비했습니다.
