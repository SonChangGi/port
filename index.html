Analysis Flow

핵심 입력과 결과 섹션을 카드형 흐름으로 먼저 보여줍니다.

01 티커·보유 주수 입력 새 ticker가 캐시에 없으면 로컬 dev에서 자동 refresh 대상이 됩니다. 02 분석 기준일 선택 해당 날짜 이전 가격·환율·수익률만 사용해 과거 보유 시나리오를 계산합니다. 03 개별 종목 비중 확인 ETF 금액은 구성종목으로 매핑하고, 미지원/잔여분은 감사 표로 분리합니다. 04 집중위험 점검 입력 종목과 선택 구성종목의 수익률 상관관계를 나란히 확인합니다.

Inputs

포트폴리오 입력

티커와 보유 주수를 입력하면 종가 통화(USD/KRW)를 기준으로 평가금액을 계산합니다. 가격이 없는 티커는 명시적으로 오류 처리됩니다.

기준일 입력값을 바꾸면 해당 날짜 이전의 종가·환율·수익률만 사용해 과거 보유 시나리오를 다시 계산합니다.

종목·보유 주수

종가 통화는 생성 JSON의 티커 메타데이터를 기본 사용하며, 기준일의 종가·환율·수익률만 분석에 사용합니다.

분석 기준일

“이 날짜에 이 보유 주수를 들고 있었다면”을 계산합니다. 종가, USD/KRW, 상관관계 표본, ETF holdings 사용 가능 여부가 모두 이 날짜 기준으로 판정됩니다.

해당 기준일 가격/환율 히스토리가 JSON에 없으면 로컬 npm run dev에서 자동으로 더 긴 가격 범위 refresh를 시도합니다. historical holdings가 없으면 미래 구성종목으로 보정하지 않고 잔여 노출에 표시합니다.

Data Health

데이터 신선도와 실패 상태

정적 페이지는 생성된 JSON만 읽습니다. 최신 데이터는 refresh 스크립트 또는 GitHub Actions에서 갱신합니다.

Refresh

새 티커 종가 추가·업데이트

브라우저는 외부 금융 API를 직접 호출하지 않습니다. 한국 ETF 6자리 코드(예: 069500, 0167A0)는 .KS로 정규화하고, 새 ETF는 종가와 holdings 파싱 대상에 함께 넣습니다. 로컬 npm run dev에서는 티커 입력만으로 refresh/test가 자동 실행됩니다.

Actions에서 실행
npm run refresh:data && npm test

로컬 자동 갱신은 npm run dev로 연 페이지에서만 동작합니다. 공개 Pages는 보안상 브라우저에서 GitHub Actions write token을 받지 않으므로, 새 티커나 기준일 히스토리 누락이 감지되면 입력값·범위·명령을 자동으로 준비하고 Actions 링크/명령 복사로 갱신합니다. 갱신 후 스냅샷 JSON과 히스토리 JSON을 다시 읽으면 기준일 종가와 환율이 보유 주수 계산에 사용됩니다.

TSLL/SNXX/0167A0/RAM ETF 갱신 입력과 자동 refresh 감지를 준비했습니다.
